Memahami Containerization untuk Pelajar PPLG

AK

Akiro

Dipublikasikan 25 April 2026

Bagikan:

Bayangkan sebuah kotak yang berisi semua yang dibutuhkan aplikasi Anda untuk berjalan — kode, runtime, library, bahkan variabel lingkungan — dan kotak itu bisa dijalankan di komputer manapun tanpa konflik. Itulah esensi containerization, dan inilah mengapa ekosistem MEeL mengadopsinya.

Bagi pelajar PPLG (Pengembangan Perangkat Lunak dan Gim), memahami containerization bukan sekadar nilai tambah di portofolio — ini adalah keterampilan fundamental yang membedakan developer junior dari yang siap kerja. Di Proyek MEeL, Docker digunakan untuk mengisolasi layanan web, database, dan tool media processing.

Apa Itu Container?

Container adalah unit perangkat lunak standar yang mengemas kode dan semua dependensinya sehingga aplikasi berjalan dengan cepat dan andal di berbagai lingkungan komputasi. Berbeda dengan Virtual Machine (VM) yang mengemulasi seluruh hardware, container berbagi kernel OS host — sehingga jauh lebih ringan.

"Di laptop Intel Core i5-6300U dengan 8GB RAM, saya bisa menjalankan 5–7 container Docker secara bersamaan tanpa lag. Coba lakukan itu dengan 5 Virtual Machine." — Catatan Pengembangan MEeL

Docker dalam Ekosistem MEeL

Proyek MEeL menggunakan Docker Compose untuk mendefinisikan dan menjalankan multi-container. Dengan satu file docker-compose.yml, seluruh stack LAMPP dapat dijalankan atau dihentikan dengan satu perintah.

# docker-compose.yml — Stack MEeL Sederhana version: '3.8' services: web: image: php:8.4-apache ports: - "80:80" volumes: - ./src:/var/www/html depends_on: - db db: image: mariadb:11 environment: MYSQL_ROOT_PASSWORD: meel_secret MYSQL_DATABASE: meel_db volumes: - db_data:/var/lib/mysql phpmyadmin: image: phpmyadmin:latest ports: - "8080:80" depends_on: - db volumes: db_data:

Perintah Docker yang Wajib Dikuasai

Sebagai pelajar PPLG, ada beberapa perintah Docker yang harus menjadi refleks Anda. Dimulai dari menjalankan seluruh stack, hingga memeriksa log ketika sesuatu tidak berjalan semestinya:

# Menjalankan semua service di background docker compose up -d # Melihat status container yang berjalan docker compose ps # Melihat log real-time dari service web docker compose logs -f web # Masuk ke dalam container untuk debugging docker compose exec web bash # Menghentikan semua service docker compose down

Keamanan Container di Jaringan Lokal

Salah satu keunggulan containerization di ekosistem MEeL adalah isolasi jaringan. Setiap container berjalan dalam network virtual tersendiri, sehingga layanan database tidak terekspos langsung ke jaringan fisik (LAN) tanpa konfigurasi eksplisit. Ini adalah lapisan keamanan tambahan yang tidak perlu dikonfigurasi secara manual.

Kesimpulan

Containerization bukan tren sementara — ini adalah cara industri membangun dan mendeploy perangkat lunak modern. Dengan mempelajarinya melalui lensa Proyek MEeL, pelajar PPLG tidak hanya memahami teori, tetapi langsung mempraktikkan prinsip DevOps yang digunakan perusahaan teknologi kelas dunia.
