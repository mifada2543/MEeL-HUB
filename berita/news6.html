Performance Improvement: Benchmark MEeL Terbaru di 2026

AK

Akiro

Dipublikasikan 20 April 2026

Bagikan:

Setelah serangkaian update pada stack LAMPP, konfigurasi Apache, dan optimasi query MariaDB, Proyek MEeL mencatat peningkatan performa yang signifikan. Data ini dikumpulkan selama 30 hari operasional di Intel Core i5-6300U dengan 8GB RAM.

Benchmark dilakukan menggunakan kombinasi Apache Benchmark (ab), sysbench untuk database, dan pengukuran manual menggunakan waktu eksekusi FFmpeg. Semua pengujian dilakukan dalam kondisi idle normal tanpa beban artifisial.

Hasil Benchmark: Web Server (Apache)

Pengujian dilakukan dengan ab -n 1000 -c 50 (1000 request, 50 concurrent). Hasilnya dibandingkan antara konfigurasi default vs konfigurasi yang dioptimalkan MEeL:

Apache Benchmark — Request Per Second

Apache Default (No OPcache)142 req/s
Apache + OPcache (PHP 8.4)487 req/s
Apache + OPcache + JIT + mod_deflate624 req/s ↑
"Dari 142 ke 624 request per detik — tanpa upgrade hardware sepeserpun. Ini adalah kekuatan konfigurasi yang benar." — Catatan Benchmark MEeL

Hasil Benchmark: Database (MariaDB)

Query database dioptimalkan melalui indexing yang tepat, query cache, dan konfigurasi InnoDB buffer pool yang disesuaikan dengan kapasitas RAM 8GB server MEeL.

# Konfigurasi MariaDB untuk MEeL — /etc/mysql/mariadb.conf.d/50-server.cnf [mysqld] # InnoDB Buffer Pool — alokasikan ~60% dari total RAM innodb_buffer_pool_size = 4G innodb_log_file_size = 256M # Query Cache (efektif untuk beban baca tinggi) query_cache_type = 1 query_cache_size = 64M query_cache_limit = 2M # Koneksi maksimum untuk LAN max_connections = 100

Hasil Benchmark: FFmpeg Transcoding

Pengujian transcoding video 1080p (H.264) ke 720p (H.265/HEVC) menggunakan libx265. Dengan optimasi threading dan preset yang tepat, waktu proses turun drastis:

FFmpeg — Waktu Transcoding (menit, lebih rendah = lebih baik)

FFmpeg Default (single-thread, preset slow)12.4 menit
FFmpeg Multi-thread (4 core, preset medium)4.8 menit
FFmpeg Multi-thread (preset fast, CRF 28)2.9 menit ↑

Kesimpulan

Data ini membuktikan premis inti Proyek MEeL: hardware lama yang dikonfigurasi dengan cermat bisa menandingi — bahkan melampaui — hardware baru yang berjalan dengan konfigurasi default. Di tahun 2026, pengetahuan tentang optimasi sistem adalah aset yang nilainya lebih tinggi dari budget pembelian server baru.
